friend_function

We know that in c++, the data member are only accessed with the member function of the same class which is called data hiding.
But there is a situstion there are two classes and it have different data members. These two classes data member are access through a single function that function is know as the friend function.
Friend functionis non member function of any class. These function access private data of those class where it is introduced as a friend function.
But there a disadvantage with friend function, c++ main concept is data hiding that outerare not to allowed to access the data. But the friend function accessing the data outside.

RULES FOR FRIEND FUNCTION
  • friend function name is whatever it may be and it is not relared to class name are function name.
  • we can declare friend function anywhere we want in class like at public, private, protected. Because
  • friend function declaration sholud be inside the class with friend keyword.
  • friend function should define outside the class. And don't use friend keyword, friend function can introduce any no.of classes.
  • it have object as arguments, so we can access the data outside.
  • It is called like a normal function and don't use the object name followed by (.) membership operator.
  • friend function is non-member function. Then we access the private data by using the object name followed by (.) operator in the friend function.
