mutilple inheritance


multiple inheritance is the deriving a derived class from more than one base class is called multiple inheritance.

Suppose we have two base classes A and B ane we derived a sub class C from the both super class A and B. Then the sub class C contain the properties of both the super classes.


  • Let's have a talk about protected members, in multiple inheritance we use the protected access specifier. We know that the data member are private in the class and these members are not allowed to access the outside the class. Suppose if we want to use the data members in both classes(base & derived) then we go for protected.
  • If the data members are in protected mood they can access in that class and immediately next class i.e immediately next derived class. This is the key point we have to consider. And futher visibility is depends on the visibility mood.
  • If the derived class is in public mood. Then the super class protected members are once again protected in derived class and further inheritance is allowed.
  • If the derived class is in the private mood, Then the super class protected member are private in the derived class and these are access by the public member function of that derived class abd further inheritance is not allowed.
